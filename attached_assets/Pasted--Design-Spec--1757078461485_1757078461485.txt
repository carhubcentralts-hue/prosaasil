UI Design Spec — Admin & Business Home + Sidebar (RTL + Mobile-first)

0) Design principles
	•	Mobile-first, RTL by default.
	•	8px spacing system (4px רק כשצריך).
	•	Accessible by design (contrast, focus, keyboard).
	•	Clean, enterprise-like, inspired by Monday/Linear.

⸻

1) Foundations (Design Tokens)

Typography
	•	Brand fonts:
	•	Hebrew: Heebo (400/500/600/700)
	•	Numbers/Latin: Inter (400/600, tabular numerals for KPIs)
	•	Base size: 16px (tailwind text-base)
	•	Scale:
	•	Display: text-3xl / font-semibold (mobile), text-4xl (desktop)
	•	H1: text-2xl / font-semibold
	•	H2: text-xl / font-semibold
	•	H3: text-lg / font-medium
	•	Body: text-base / text-slate-800
	•	Helper: text-sm text-slate-500
	•	Numbers in KPIs: wrap with .tabular-nums (Inter).

Tailwind v4 CSS (once, global):

@import "tailwindcss";
:root{
  --brand: #2563eb;      /* primary blue-600 */
  --brand-600:#2563eb;
  --brand-500:#3b82f6;
  --brand-700:#1d4ed8;
  --ok:#16a34a;          /* success */
  --warn:#f59e0b;        /* warning */
  --danger:#ef4444;      /* error */
  --bg:#f8fafc;          /* slate-50 */
  --card:#ffffff;
  --text:#0f172a;        /* slate-900 */
  --muted:#64748b;       /* slate-500 */
  --ring:#93c5fd;        /* blue-300 */
}
.tabular-nums{ font-feature-settings:"tnum" 1; }

Color Palette
	•	Primary: --brand (#2563EB)
	•	Neutrals: slate (#0f172a..#e2e8f0)
	•	Semantic: success #16a34a, warning #f59e0b, danger #ef4444
	•	Gradients (hero chips/headers): from-violet-600 to-blue-600.

Radii & Shadows
	•	Cards: rounded-2xl + shadow-sm + border border-slate-200/60
	•	Inputs/Buttons: rounded-xl
	•	Modals/Drawers: rounded-2xl
	•	Focus: ring-2 ring-[var(--ring)] ring-offset-1

Spacing
	•	Grid: 8px (Tailwind: p-2, p-3, p-4, gap-4, gap-6)

⸻

2) Global layout & RTL
	•	HTML root: <html dir="rtl" lang="he">
	•	Container: mx-auto max-w-[1200px] px-4 md:px-6
	•	Page header: title + actions (pill buttons) in a responsive grid.
	•	Numbers that must be LTR: wrap with <span dir="ltr">052-…</span>.

⸻

3) Navigation

Sidebar (Desktop ≥ md)
	•	Right-aligned (RTL), width w-72.
	•	Structure:
	•	Top: Logo + app name
	•	User capsule: avatar initial + role + tenant
	•	Nav list groups (no ARIA menubar; just <nav> with list)
	•	Item (active): bg-slate-100 text-slate-900 border border-slate-200
	•	Icon set: Lucide/Heroicons outline (keep stroke width consistent).
	•	Hover/focus: subtle translate-x / ring.

Drawer (Mobile < md)
	•	Toggle button in header (aria-expanded, aria-controls).
	•	Slide-in from right (translate-x-full → translate-x-0)
	•	Focus trap inside drawer; Esc closes; return focus to toggle.
	•	Optional Bottom Navigation (3–5 items): Leads, WhatsApp, Calls, Calendar.

Sidebar items (as your screenshot): Overview, Leads, WhatsApp, Calls, CRM, Payments & Contracts, Business Manager, Users, Settings, Calendar.
Only Overview routes are live now; others open a “Coming soon” sheet (no console errors).

⸻

4) Components (visual spec)

Buttons
	•	Primary: bg-[var(--brand)] text-white hover:bg-[var(--brand-700)] disabled:opacity-50 rounded-xl px-5 py-2.5
	•	Secondary: bg-white border border-slate-200 text-slate-700 hover:bg-slate-50
	•	Ghost: hover:bg-slate-100 text-slate-700
	•	Loading: spinner inline (12px), aria-busy="true"

Inputs
	•	rounded-xl border border-slate-300 bg-white px-4 py-3 focus:outline-none focus:ring-2 focus:ring-[var(--ring)]
	•	Error: border-[var(--danger)] text-[var(--danger)] + help text text-sm.
	•	Password visibility toggle with aria-pressed.

Cards
	•	Container: rounded-2xl bg-[var(--card)] shadow-sm border border-slate-200
	•	KPI card:
	•	Header: label text-sm text-[var(--muted)]
	•	Number: text-3xl md:text-4xl font-semibold tabular-nums
	•	Delta badge: small pill (▲/▼ + color)
	•	Footer hint: text-xs text-slate-500

Badges (status/providers)
	•	Base: inline-flex items-center gap-1 rounded-full px-2.5 py-1 text-xs
	•	Success: bg-green-50 text-green-700 ring-1 ring-green-200
	•	Warning: bg-amber-50 text-amber-700 ring-1 ring-amber-200
	•	Danger: bg-red-50 text-red-700 ring-1 ring-red-200
	•	Neutral: bg-slate-100 text-slate-700 ring-1 ring-slate-300

Skeletons
	•	animate-pulse bg-slate-100 rounded-md h-4 with varying widths

Toasts
	•	Bottom (RTL → right): stacked, auto-dismiss 6s, semantic colors.

⸻

5) Pages (content + layout)

A) Login / Forgot / Reset (polish for mobile)
	•	Header card with logo-initial (rounded-2xl, gradient), title, subtitle.
	•	Form card: max-w-md mx-auto p-6 md:p-8 rounded-2xl shadow-sm bg-white border
	•	Fields: Email, Password; remember me checkbox.
	•	Actions: Login (full width), link “שכחת סיסמה?”
	•	Forgot: email input → success screen (never leak existence).
	•	Reset: token param, new password + confirm, password meter (client-side only), submit.

B) Admin Home (/app/admin/overview)
	•	Header: “מנהל מערכת” + quick filters (Today / 7d).
	•	Row 1 (grid-cols-1 md:grid-cols-3):
	•	Provider Status card:
	•	Twilio (Inbound/Outbound), Baileys, DB: badges with up/partial/down + latency (stt/ai/tts).
	•	KPI: Businesses (Total / Active)
	•	KPI: Unread total
	•	Row 2 (grid-cols-1 md:grid-cols-2):
	•	KPI: WhatsApp today + last7d sparkline
	•	KPI: Calls today + avg handle (sec)
	•	Row 3:
	•	Recent Activity table/list (20 items): time, type (whatsapp/call), tenant, preview, “Open”.

Data (JSON):
	•	GET /api/status → provider badges & latency
	•	GET /api/dashboard/stats → KPIs (admin scope)
	•	GET /api/dashboard/activity → recent activity

C) Business Home (/app/business/overview)
	•	Header: “שי דירות ומשרדים” + Today / 7d switch.
	•	Row 1:
	•	Provider Status (same component)
	•	KPI: New leads today
	•	KPI: Active leads
	•	Row 2:
	•	KPI: Unread (WA)
	•	KPI: Calls today
	•	Row 3:
	•	Quick actions: Leads / WhatsApp / Calls / Calendar (icon buttons, large hit area)
	•	Recent tenant activity (same schema, filtered by tenant)

⸻

6) Accessibility & micro-interactions
	•	Focus: visible ring on all interactive elements.
	•	Keyboard: Tab order logical; Esc closes drawer; return focus to trigger.
	•	Toast with role="status"; error region role="alert".
	•	Loading: skeletons in KPI/cards; never full-page spinner.

⸻

7) Implementation notes (Tailwind/React)
	•	Create a cn() helper (clsx + tailwind-merge) for safe class merges.
	•	Use component wrappers: <Card>, <StatCard>, <Badge>, <KpiGrid>, <ProviderStatus>, <RecentActivity>.
	•	Enforce one global CSS import with @import "tailwindcss";
	•	Don’t use dynamic Tailwind classes like bg-${color}-500 without safelist.

⸻

8) Data contracts to consume (unchanged)
	•	GET /api/auth/me → { user:{id,email,role}, tenant:{id,name} }
	•	GET /api/status → { twilio:{up}, baileys:{up}, db:{up}, latency:{stt,ai,tts} }
	•	GET /api/dashboard/stats →
{ calls:{today,last7d,avgHandleSec}, whatsapp:{today,last7d,unread}, revenue:{thisMonth,ytd} }
	•	GET /api/dashboard/activity →
{ items:[{ts,type:'whatsapp|call',tenant,preview,provider}] }

⸻

9) Definition of Done (visual)
	•	Sidebar responsive (desktop fixed, mobile drawer + bottom nav), RTL, accessible.
	•	Admin & Business Home match the content layout above (cards, badges, activity, skeletons).
	•	Login/Forgot/Reset fully polished on mobile.
	•	No console errors; assets served with correct MIME; crisp UI (spacing/typography/colors consistent).

⸻

Tiny QA checklist (dev can tick)
	•	<html dir="rtl"> set
	•	Sidebar focus trap + Esc close (mobile)
	•	Buttons/inputs hit area ≥ 44×44px
	•	KPI numbers use .tabular-nums (Inter)
	•	Provider badges show correct states from /api/status
	•	Activity list truncates previews and shows time in ISO-local
	•	No dynamic Tailwind class names without safelist
