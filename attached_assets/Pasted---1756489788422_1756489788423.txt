# 2) UX/UI – בדיוק איך זה צריך להיראות ולהרגיש

### עקרונות

* **שכבות**: רקע חי (גרדיאנטים מטושטשים), **Glass Card** (ריבוע לבן שקוף עם blur+ring), פאנל מותג נפרד בדסקטופ, טופס ממורכז במובייל.
* **טיפוגרפיה**: Heebo 400/600/700, סולם גודל 14–18, קונטרסט AA.
* **Micro-interactions**: מעבר ריחוף עדין, הופעת טוסט/שגיאה עם fade+slide, כפתור “הצג סיסמה”.
* **נגישות**: label לכל אינפוט, focus-ring, מקשי Tab/Enter/Esc לדיאלוגים, aria-live לשגיאות. ([W3C][11])

### היררכיית מסכים

1. **Login**

   * שדות: אימייל (`dir="ltr"`, `autoComplete=email`), סיסמה (`autoComplete=current-password`), “זכור אותי”.
   * קישורים: “שכחתי סיסמה” → `/auth/forgot`, “דף הבית”.
   * כפתור ראשי: גרדיאנט (brand→accent), מצב `loading`.
   * ולידציה מיידית (אימייל חוקי, סיסמה ≥ 8).
   * ניתוב אחרי 200: `role ∈ {admin, superadmin}` → `/app/admin`, אחרת `/app/biz`.

2. **Forgot**

   * שדה אימייל, כפתור “שלח קישור איפוס”.
   * תמיד מחזיר הודעת הצלחה ניטרלית (“אם האימייל קיים…”).

3. **Reset**

   * שני שדות סיסמה, חיווי התאמה, שמירה → Toast “עודכן” → הפניה ל-Login.

### קומפוננטים/דפוסים שחובה להשתמש

* **Card מזוגג**: `bg-white/70 backdrop-blur-xl ring-1 ring-black/5 rounded-3xl shadow-2xl`.
* **Gradient CTA**: `bg-gradient-to-l from-[#6366f1] to-[#22d3ee]`.
* **Icon fields**: אייקון מעטפה/מנעול בצד השדה (RTL—האייקון בצד ימין).
* **Panel מותג** (בדסקטופ): לוגו “שי”, כותרת “שי דירות ומשרדים”, 3 בולטים של יתרונות.
* \*\* אנימציות Motion\*\*: fade-in לכרטיס, hover-scale קל לכפתור, stagger לרכיבים.

> בקש מהבוט: “בנה Layout דו-טורי בדסקטופ (grid cols-2), כרטיס התחברות ממורכז, פאנל מותג בצד. במובייל—טור אחד. הוסף gradient blobs ברקע עם blur ו-pulse.”

### סטיילינג/טוקנים (Tailwind v4\*)

* צבעי מותג: brand=neutral-900, accent=indigo-500/cyan-400.
* רדיוס: `rounded-2xl/3xl`.
* צל: `shadow-2xl` (כרטיס), `shadow-lg` (CTA).
* ריווח: `py-3 px-4` בשדות; טפסים `space-y-3/4`.
* **Min tap**: כל כפתור/שדה `min-h-[44px]`.

> Tailwind 4 פרודקשן (שיטת tokens, תועלות שדרוג): שמור עדכוני v4.1. ([GitHub][5], [endoflife.date][6])

# 3) חוקי עבודה לבוט (כדי שלא “יתפרק” ויזואלית)

## 3.1 תבנית משימה (חובה לציית)

**מטרה**: לבנות `/auth/login`, `/auth/forgot`, `/auth/reset` ב-React 19 + Tailwind 4.1, בסגנון פרימיום.
**כללי זהב**:

1. **אל תיצור פרויקטים כפולים**: ספרייה יחידה `auth-frontend/` עם `src/pages/...`.
2. **אל תשנה את דשבורדי Flask**; תנתב אליהם אחרי login.
3. **השתמש ב-Heebo** דרך `<link>` בגולגל פונטס. ([Google Fonts][10])
4. **אנימציות** רק עם Motion; אל תכניס ספריות UI כבדות. ([Motion][7])
5. **דיאלוג/Toast**: אם צריך רכיב—השתמש ב-Radix/shadcn (קוד פתוח להתאמה). ([Radix UI][8], [Shadcn][9])
6. **נגישות**: עמידה ב-WAI-ARIA לטפסים/דיאלוג. ([W3C][11])
7. **אין CSS חיצוני**—Tailwind בלבד; אין inline styles כבדים מלבד רקע גרדיאנטים.

## 3.2 צעדים לביצוע (סדר מחייב)

1. **תשתית**: אתחל פרויקט Vite React (JSX), התקן Tailwind 4.1, הוסף Heebo, התקן Motion. ([Yarn][4], [GitHub][5], [Motion][7])
2. **תצורה**: `index.html` עם `dir="rtl"`, `<meta viewport-fit=cover>`, `<link Heebo>`. ([Google Fonts][10])
3. **Router**: `BrowserRouter basename="/auth"` עם ראוטים לשלושת המסכים.
4. **Theme Tokens**: הגדר צבעי brand/accent, רדיוסים ו-shadows ברמת Tailwind config. ([GitHub][5])
5. **Layout**: מסך Login בדו-טורי (דסקטופ)/טור יחיד (מובייל), עם רקעי gradient+blur.
6. **טופס**: שדות עם אייקונים, ולידציה, מצב loading, כפתור CTA גרדיאנט.
7. **אינטגרציה**: כל הקריאות ל-`VITE_API_BASE` עם `credentials:"include"`.
8. **בדיקות**:

   * רספונסיביות (320px–1440px).
   * נגישות (Tab order, Enter submit, צבעי קונטרסט). ([W3C][11])
   * “שכחתי סיסמה” ו-“Reset” זורמים עד הסוף.
   * ניתוב אחרי Login לפי `user.role`.

## 3.3 ניקוי כפילויות בצד השרת (מאוד חשוב)

* **מסלול יחיד ל-/auth**: ודא שאינך מגיש גם Jinja וגם React לאותו נתיב.
* מחק/נטרל **templates ישנים של התחברות** ב-Flask, כדי שלא “יתגנבו” בדפדוף.
* **סטטיק יחיד**: אם Flask מגיש `/static`, אל תיצור תיקיית סטטיק נוספת עם אותן תמונות/פונטים.
* CORS/Session: ה-API חייב להחזיר/לקבל cookies (`credentials:true`).
* **Redirects יחידים**: אחרי login—*תמיד* הפניה ל-`/app/admin` או `/app/biz`; אין קישורי placeholder.

## 3.4 “Definition of Done” – קבלות ויזואליות

* נראה *אפליקציה*: רקע גרדיאנט מטושטש, כרטיס glass, אנימציות עדינות.
* טופס “חי”: אייקונים, טעינה, שגיאות, “הצג/הסתר סיסמה”.
* מובייל: בלי גלילה אופקית, טפסים בני-שימוש, tap targets ≥ 44px.
* נגישות: labels, focus-ring, aria-live להודעות, מקשי מקלדת. ([W3C][11])
* מותג: “שי דירות ומשרדים” + Heebo + צבעי המותג. ([Google Fonts][10])
